## Основы работы с `pytest`

`pytest` - это самая популярная библиотека для написания тестов.

### Установка

```shell
pip install pytest
```

### Обнаружение тестов

Тесты ищутся по следующим правилам:
- рекурсивно просматриваются папки
- в папках ищутся файлы с именами `test_*.py` или `*_test.py`
- в этих файлах ищутся объекты для тестирования, которыми могут быть:
  - функциями с префиксом `test`
  - функции и методы с префиксом `test` внутри классов с префиксом `Test`

### Запуск тестов

Посмотреть все возсожные для запуска тесты можно командой:

```shell
pytest --collect-only -q
```
- `--collect-only` указывает, что надо собрать тесты без запуска
- `-q` более компактный вывод (*quite mode*), противоположность - флаг `-v`

Запустим все тесты, описанные в файле `src/test_sample.py`:

```shell
pytest demo_app/test_sample.py -v
```
- `-v` - наоборот, выводить больше информации

Запуск конкретного теста:
```shell
pytest demo_app/test_sample.py::test_sum
```

Запустим тесты класс из файла `src/test_class.py`:
```shell
pytest demo_app/test_class.py -v
```

Обратим внимание на тест `TestClassShared::test_two`. Ошибка связана с тем, что объект 
класса, в котором описаны тесты, создаётся заново для каждого теста. Однако при этом атрибуты класса 
остаются общими (`TestClassShared::test_three` и `TestClassShared::test_four`).


Организация тестов в классы полезна по следующим причинам:
- логическая организация тестов
- определение фикстур, видимых только тестам внутри класса
- использование маркеров `pytest.mark` на класс применяется ко всем тестам внутри

### pytest.raises

Для того, чтобы проверить в тесте выброс исключения используется `pytest.raises`:
```python
import pytest

def parse_age(s: str) -> int:
    if not s.isdigit():
        raise ValueError("age must be a number")
    return int(s)

def test_parse_age_rejects_non_digits():
    with pytest.raises(ValueError):
        parse_age("xx")
```
